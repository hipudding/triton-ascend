# 社区技术会议

Triton-Ascend 定期召开社区技术会议，作为项目治理的核心决策机制。技术方向、大型方案评审、版本发布等重大事项均在技术会议上进行讨论和决策，由所有 `Maintainer` 投票产生。会议对所有社区成员公开，任何人均可参加讨论，但决策和投票仅由 `Maintainer` 完成。

## 会议内容

- 技术方向讨论与决策
- 大型方案（RFC）评审与投票
- 版本发布计划与时间线
- 社区 Issue 和 PR 的优先级排序
- 贡献者提名与任免讨论

## 投票规则

- 未参会的 `Maintainer` 需在会议后 3 个工作日内反馈投票意见，逾期未投票视为弃权
- 投票以参与投票的 `Maintainer` 过半数同意即为通过（弃权不计入票数）
- 投票结果记录在会议纪要中

## 决策流程

### 常规决策

日常开发中的常规决策（如 `Bug` 修复、小功能改进）由相关模块的 `Maintainer` 在 `Issue` 或 `PR` 中审核决定，无需上会。

### 重大决策

以下事项视为重大决策，需在社区技术会议上由 `Maintainer` 集体讨论决定：

- 架构层面的重大变更
- `API` 的破坏性变更
- 新增核心功能或移除已有功能
- 第三方依赖的重大升级
- 版本发布计划

重大决策的流程：

1. 提交 `RFC`（Request for Comments），描述变更内容、影响范围和实施方案
2. 在技术会议上进行讨论
3. `Maintainer` 共同决策
